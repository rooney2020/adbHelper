# ADB Helper 项目原则声明

**版本**: v1.0.0 | **批准日期**: 2026-05-12 | **最后修订**: 2026-05-12

## 核心原则

### P1. [架构] Capability 优先执行

所有 adb 操作必须先完成设备识别、命令适用性判断和替代链路选择；禁止 UI 层直接拼接命令后盲执行。

**验证方法**: 抽查任一命令执行链路，确认先经过 capability-engine，再进入 adb-runner。

### P2. [安全] 前后端最小暴露面

Renderer 必须运行在 `contextIsolation` 开启且 `nodeIntegration` 关闭的环境中；禁止页面直接访问系统命令或任意文件系统。

**验证方法**: 检查 Electron 启动配置与 preload 白名单 API，确认系统能力只通过受控 IPC 暴露。

### P3. [架构] 原文不可丢失

任何文本型命令即使结构化解析失败，也必须保留原始输出、退出码、stderr 摘要和导出能力。

**验证方法**: 对解析失败样例执行测试，确认结果页仍可回退到原文视图并导出。

### P4. [代码质量] 全量命令可导航而非平铺

“支持全部 adb 命令”必须通过分类目录、搜索、收藏、最近使用和原始命令模式组合实现，禁止用静态按钮堆叠代替信息架构设计。

**验证方法**: 检查命令入口设计，确认存在目录索引、搜索与收藏机制，而不是简单按钮墙。

### P5. [性能] 大结果可读性优先

对于 dumpsys、logcat、am、cmd 等大输出，结果展示层必须优先实现搜索、高亮、折叠、分段和虚拟化渲染。

**验证方法**: 对大文本样本执行性能测试，确认渲染和搜索满足计划中的性能指标。
