# 组件映射

## 映射原则

- 原型组件名称应和实现组件职责一一对应。
- 优先复用现有组件，禁止在实现阶段无依据改换控件类型。
- 对高风险控件必须明确“默认实现 + 样式策略 + 风险点”。

## 组件映射表

| 原型组件 | 所在页面 | 技术实现 | 样式策略 | 数据来源 | 状态要求 | 风险备注 |
|---------|---------|---------|---------|---------|---------|---------|
| 标题栏设备入口 | 主工作台 | React Header Action | 右侧摘要 + 触发按钮 | 当前选中设备 | normal/hover/active | 收起态必须一直可见，不得占用正文高度 |
| 设备选择 popup | 主工作台 | React Popover / Floating Panel | 标题栏右侧浮层 + 设备列表 + 摘要 | `device.list()` | collapsed/expanded/empty | 展开后不得压缩主工作台正文 |
| 命令体系树 | 主工作台 | React Tree / 分组导航 | 左侧分组树 + 当前项高亮 | 本地命令元数据索引 | normal/hover/selected/disabled | 层级过深时需保持可扫描性 |
| 顶部概览条 | 主工作台 | React Layout 组件 | 渐变背景 + 玻璃面板 | 当前设备、当前结果 | normal/loading/error | 需适配不同窗口宽度 |
| 设备卡片列表 | 主工作台 | React 列表 + 自定义卡片 | 选中态淡蓝高亮 | `device.list()` | normal/hover/active/empty/error | 多设备时滚动体验关键 |
| 二级命令导航项 | 主工作台 | React 列表 + 分类过滤 | 中部列表 + 单开详情 + 当前项高亮 | `command.search()` | normal/hover/selected/disabled | 全量命令规模大，虚拟化可能需要 |
| 当前命令详情卡 | 主工作台 | React Summary Card | 白底卡片 + 徽章 + 参数表单分组 | 选中命令元数据 | normal/empty/error | 必须与原始命令编辑器保持同步 |
| 参数配置表单 | 主工作台 | React Form + 本地状态 | 必填/可选分组 + 紧凑输入控件 | 选中命令 `params` | normal/focus/error/disabled | 输入变化必须实时驱动命令拼装 |
| 适配洞察信息按钮 | 主工作台 | React Icon Button | 小型圆形按钮 + 低干扰高亮 | 当前命令 capability 摘要 | normal/hover/focus | 必须靠近当前命令标题，不得占用独立栏位 |
| 原始命令编辑器 | 主工作台 | React Textarea / 代码编辑器封装 | mono 字体 + 边框高亮 | 本地表单状态 | focus/error/disabled | 长命令行折行策略需明确 |
| 左右拖拽分隔条 | 主工作台 | React Split Pane / 自定义拖拽条 | 细线低干扰 + hover 可见 | 布局状态 | idle/dragging | 只改变宽度，不改变区域顺序 |
| 上下拖拽分隔条 | 主工作台 | React Split Pane / 自定义拖拽条 | 细线低干扰 + hover 可见 | 布局状态 | idle/dragging | 只改变执行区高度，不改变页面整体滚动 |
| 结果 Tab 栏 | 主工作台 | Tabs 组件 | 胶囊标签 + 下划线高亮 | 本地状态 | normal/hover/active/focus | 需保证键盘可访问性 |
| 结构化树视图 | 主工作台 | 自定义 Tree + Detail Pane | 轻卡片 + 选中行高亮 | `parse_result()` | normal/active/empty/loading/error | 长树节点和虚拟滚动是风险点 |
| 高风险确认弹窗 | 弹层 | Dialog 组件 | 大圆角、强遮罩、红色确认按钮 | 命令元数据 + 当前设备 | normal/loading/error | 不允许替换为浏览器原生 confirm |
| 适配洞察弹窗 | 弹层 | Dialog 组件 | 居中浮层 + 信息分组 + 快照操作区 | `capability.report()` + `history.list()` | normal/loading/error | 不允许恢复成常驻侧栏占用工作台空间 |

## 自定义组件清单

| 组件 | 为什么不能直接用原生控件 | 计划实现方式 |
|------|----------------------|-------------|
| 二级命令导航项 | 原生列表无法同时承载命令摘要、筛选与当前项状态 | 自定义 React Tree Item 组件 |
| 参数配置表单 | 原生静态表单无法自动联动 raw 命令与命令模板 | 自定义受控表单组件 |
| 结构化结果浏览器 | 需要树视图、明细面板、原文切换和高亮搜索 | 自定义结果工作区组件 |
| 高风险确认弹窗 | 需要统一样式、遮罩、信息摘要和按钮层级 | 自定义 Dialog + Portal |
| 适配洞察弹窗 | 需要从命令详情按钮就近呼出，并复用统一弹层样式 | 自定义 Dialog + Portal |

## 组件复用与禁区

- 必须复用的组件：按钮、输入框、徽章、Tab、Dialog、Panel 容器。
- 禁止私自替换的组件：高风险确认弹窗、适配洞察弹窗、结果 Tab、设备卡片、二级命令导航项、当前命令详情卡、参数配置表单。
- 允许平台差异化处理的组件：滚动条、窗口标题栏区域、文件导出选择器。

## 质量联动

- 哪些组件必须进入原型差异审查：标题栏设备入口、设备选择 popup、设备卡片、二级命令导航项、当前命令详情卡、参数配置表单、适配洞察信息按钮、适配洞察弹窗、结构化结果浏览器、高风险确认弹窗、拖拽分隔条。
- 哪些组件必须进入真实交互回放：设备切换、一级分类切换、参数输入驱动命令变化、原始命令执行、结果 Tab、导出按钮、风险确认流程。
