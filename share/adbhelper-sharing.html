第二页 / 痛点与目标

为什么需要 ADB Helper

adb 命令数量庞大、参数复杂、不同 ROM 和 Android 版本行为不一致——开发与测试人员每天在记忆命令、排查兼容性问题和解析原始输出上耗费大量时间。

01 命令记忆成本高

adb 涵盖 device / shell / am / pm / dumpsys / settings / logcat / input / wm 等数十个子命令族，参数和 flag 组合数以百计。每次操作都要查文档、试参数，效率极低。

02 ROM 兼容性不透明

同一命令在 AOSP、MIUI、ColorOS、OneUI 上行为可能完全不同。厂商定制 ROM 可能删除或改写标准子命令。用户只能盲执行再观察报错，没有预检机制。

03 原始输出难以阅读

dumpsys 动辄数万行纯文本，堆栈、字段嵌套、Key-Value 混排——快速定位关键信息全靠肉眼 + grep，没有结构化解析、高亮、折叠和对比能力。

04 缺少工作流整合

Logcat 筛选、截屏标注、性能监控、Monkey 测试各自为战，结果散落在不同终端窗口，无法关联查看、对比分析和一键导出为测试报告。

第三页 / 核心设计理念

三大设计原则驱动整个工具

不是简单把命令行套一个 GUI 壳——而是从根本上重新思考"如何在 GUI 中有意义地承载全部 adb 能力"。

原则 1 Capability 优先执行

每条命令执行前先探测设备能力（Android 版本、ROM 特征、命令是否可用），输出"支持 / 受限 / 不支持"三种状态，并在不可用时自动给出降级方案和替代命令。禁止 UI 直接盲执行底层命令。

原则 2 全量可导航

"全部 adb 命令"不能靠堆叠几百个按钮承载。采用"分类目录 + 搜索 + 收藏 + 最近使用 + 原始命令兜底"的组合方案，让用户在 13 个分类中迅速定位目标命令，未模板化的操作也能直接输入执行。

原则 3 结果双视图

所有文本型命令必须同时保留"原始输出"和"结构化渲染"两种视图，解析失败时禁止丢弃原始文本。支持关键词搜索高亮、任意两条历史记录逐行 diff 对比、Markdown / TXT / JSON 三种格式导出。

安全隔离：Renderer（React）禁止直接访问 Node 能力或系统命令，所有执行请求经 Electron 主进程白名单 IPC 和 Python 后端适配层。高风险写操作触发二次确认弹窗。

第六页 / 命令目录

13 个分类 · 200+ 模板命令

按 adb 子命令族划分为 13 个功能组，每个命令附带概要说明、实况命令预览、风险等级和降级方案。支持搜索、收藏和原始命令模式。

应用管理 (am) 包管理器 (pm) 键鼠模拟 (input) 设备信息 (device) Shell 脚本 (shell) 窗口管理 (wm) 系统设置 (settings) 日志系统 (logcat) 属性查询 (property) 截屏录屏 (screenshots) 权限管理 (permission) 系统状态 (dumpsys) 连接管理 (connectivity)
智能表单 参数化执行

每个命令模板定义参数 schema（类型、必填、默认值），渲染为结构化输入表单。用户只需填写关键参数即可自动拼装完整 adb 命令，无需记忆 flag 语法。

风险分级 写操作二次确认

每条命令标注"查看型 / 写操作"类型和"低 / 中 / 高"风险等级。高风险写操作触发二次确认弹窗，列出将要执行的完整命令和潜在影响，防止误操作。

Capability 探测 执行前兼容预检

执行前自动探测当前设备是否支持该命令。输出"支持 / 受限 / 不支持"三种状态，受限时提示预期行为差异，不支持时自动给出替代命令和降级方案。

原始命令模式 模板化之外的兜底

未纳入模板的命令可直接输入完整原始 adb 子命令执行。结果统一进入结构化渲染和 diff 对比流程，确保"全量 adb 命令可导航"原则不被模板化边界限制。

第七页 / Logcat 日志捕获

实时流式捕获 + 多维筛选 + 自动导出

区别于传统 "adb logcat > file" 的原始使用方式，ADB Helper 提供可视化筛选、进程关联和结构化导出。

多维筛选 message / tag / PID / TID / package 组合规则

支持 AND / OR 逻辑组合多条过滤规则，每条规则独立指定过滤字段（message 内容 / tag / PID / TID / 包名）和目标值。同时支持正则模式搜索和日志级别快速过滤（V/D/I/W/E/F）。

进程关联 自动解析 PID → 包名映射

从设备 ps 输出自动构建 PID 到包名的映射表，日志条目自动填充 packageName 字段。从传统"看 PID 猜包名"升级为"一目了然知道每条日志来自哪个应用"。

自动化导出 轮转分片 + 一键 ZIP 归档

可配置单文件大小上限（MB）和输出目录，超出上限自动轮转到新文件。停止捕获后一键将当前轮转目录下所有日志 ZIP 打包下载，文件名带时间戳，方便标注和归档。

性能设计：日志行数可达 20000+ 行时仍保持流畅渲染——采用虚拟滚动（virtual scroll with overscan=20），固定行高 28px，折行行高 80px。流式捕获过程自动丢弃超出 buffer limit 的旧行，保证内存可控。

搜索高亮 全文搜索 + 匹配计数

在当前输出结果中搜索关键词，高亮所有匹配项并显示匹配次数。支持大小写不敏感匹配，搜索词自动转义正则特殊字符。

Diff 对比 逐行差异分析

在左侧和右侧分别选择两条历史记录，生成逐行 diff 视图：same（相同）/ changed（变更）/ added（新增）/ removed（删除）四种差异类型以不同背景色区分。

一键导出 Markdown / TXT / JSON 三格式

每条输出结果可导出为 Markdown（code block 格式）、纯文本或 JSON 文件。下载文件名自动带时间戳和设备名，方便归档和分享给团队成员。

历史回溯 完整的执行记录

500 条历史记录可供回看，每条包含：设备名、命令标题、完整执行命令、stdout、stderr、状态、耗时、时间戳。支持搜索过滤和时间排序。

覆盖性能诊断、电池分析、启动耗时、Activity 栈、窗口层级、SurfaceFlinger 状态等核心场景。

性能诊断 CPU / 内存 / FPS / 电量

cpuinfo / meminfo / batterystats / gfxinfo 一键查询，自动提取关键指标并以折线图可视化。

Activity 栈 前台/后台栈快照

第十页 / 备份与恢复

分区自动备份 · 版本化目录 · 一键恢复

针对嵌入式 Android 系统（如车机）开发场景，提供完整的备份、版本管理和恢复工作流。

分区备份 多分区并行 pull

配置多个备份分区路径（如 /system /vendor /product 等），一键并行 pull 到本地版本化目录。自动按设备版本号创建子目录，多版本各自独立。

Root 自动化 root → remount → adb root 链式操作

恢复操作前自动完成 root -> adb disable-verity -> reboot -> root -> remount 全链路。后台记录每一步的执行状态和耗时，失败时精准定位断点。

版本回滚 选择备份版本 → 一键恢复

从已备份的多个版本中选择目标版本，自动执行 push + reboot 流程。每个分区的推送结果独立显示，异常分区单独标红。

版本识别：备份版本通过 ro.build.version.release 等属性自动识别。备份目录命名格式为 "版本名-build_fingerprint"，确保多版本之间不会混淆。

文件系统 目录浏览 + 上传下载 + 权限操作

用户管理 多用户状态 + 限制策略

第十三页 / 投屏与布局

scrcpy 投屏 + Layout Inspector 双工具联动

投屏实时查看设备画面，Inspector 深入 UI 层级——两者互补，覆盖视觉调试全流程。

scrcpy 集成 多 Display + 位置持久化

自动检测 scrcpy 是否可用。列出设备所有 Display（物理屏 / 虚拟屏），一键启动指定 Display 的投屏。窗口位置和最大尺寸自动持久化，下次启动恢复。

通过 adb uiautomator dump 获取当前页面的 UI 节点树，展示控件层级、坐标范围、文本内容和资源 ID。可配合按键模拟的坐标点击和滑动操作使用。

Winscope Window/SurfaceFlinger 追踪可视化

集成 Android Winscope 工具，支持加载和查看 winscope 追踪文件（window_trace / layers_trace / transaction_trace / proto_log / screen_recording 等）。自定义 winscope:// 协议直接打开。

开发效率 协议直通 + 本地缓存

Winscope 前端（public/winscope/）内置于 Electron 应用内，通过自定义协议直接加载，无需额外 HTTP 服务器。追踪文件可从设备一键拉取到本地。

支持监控设备整体 CPU 使用率或指定包名的单进程 CPU 使用率。数据以折线图实时绘制，时间窗口可滚动。同时显示实时 CPU 频率和核心数信息。
